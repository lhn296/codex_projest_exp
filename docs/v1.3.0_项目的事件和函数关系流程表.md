# 2. 本版核心变化

`v1.3.0` 的重点，不只是新增一个芯片驱动，而是把：

```text
ESP32 直连 GPIO 输入
```

逐步升级成：

```text
ESP32 I2C
-> driver
-> 板级 BSP
-> button_service
-> unified event queue
-> app_event_task
```

也就是说，这版的真正重点是：

- 把 `I2C` 访问能力建立起来
- 把 `XL9555` 做成可复用模板
- 把板载按键和板载控制资源接到现有项目架构里

# 8. 关键时序图

## 8.1 板载按键事件时序图

```mermaid
sequenceDiagram
    participant U as 用户按键
    participant X as XL9555
    participant ISR as bsp_button_xl9555_int_isr_handler
    participant BTN as button_service_process
    participant Q as Queue
    participant EVT as app_event_task
    participant LED as led_service

    U->>X: 按下 KEY0~KEY3
    X->>ISR: INT 拉低
    ISR->>BTN: irq_pending = true
    BTN->>X: 读取按键输入口
    BTN->>BTN: 消抖 + 状态机识别 SHORT/LONG/DOUBLE
    BTN->>Q: button_service_publish_event(btn_id,event,now_ms)
    Q->>EVT: xQueueReceive(msg)
    EVT->>EVT: 统一事件分发
    EVT->>LED: led_service_set_mode(led_id,mode)
```

## 8.2 XL9555 设备初始化时序图

```mermaid
sequenceDiagram
    participant APP as app_main_task
    participant BTN as bsp_button
    participant BSP as bsp_xl9555
    participant XL as xl9555
    participant I2C as i2c_bus

    APP->>BTN: button_service_init()
    BTN->>BSP: bsp_xl9555_init()
    BSP->>XL: xl9555_init(0x20)
    XL->>I2C: i2c_bus_init()
    XL->>I2C: i2c_bus_probe(0x20)
    BSP->>XL: 配置 KEY0~KEY3 输入
    BSP->>APP: 注册 INT(GPIO0)
```
