# v1.4.0 项目开发文档

## 1. 版本摘要

- 版本号：`v1.4.0`
- 日期：`2026-03-16`
- 平台版本：`ESP-IDF 5.5.3`
- 阶段主题：`SPI LCD Display Template`
- 版本定位：在 `v1.3.1` 的板载交互基础上，正式接入 `SPI + ST7789V LCD` 显示链，沉淀一套可复用的显示驱动与显示服务模板。

## 2. 版本目标

- 新增 `SPI` 通用驱动模板
- 新增 `lcd_st7789v` 显示驱动模板
- 新增 `bsp_lcd` 板级 LCD 适配层
- 新增 `display_service` 显示服务层
- 在 LCD 首页显示版本、阶段、LED 状态、蜂鸣器状态和最近按键事件
- 保持现有统一事件架构不变，让 LCD 成为新的输出端

## 3. 版本背景与演进关系

- `v1.3.1`
  - 完成板载四键输入
  - 完成蜂鸣器业务层
  - 完成 `XL9555` 输入输出协同
- `v1.4.0`
  - 在已有 `I2C + XL9555 + Queue + LED + BEEP` 基础上，引入 `SPI LCD`
  - 重点学习“总线模板 + 芯片驱动 + 板级适配 + 显示服务”这一条完整显示链

## 4. 本版本核心实现内容

### 4.1 SPI 通用总线层

新增 `spi_bus`，用于：

- 总线初始化
- 设备注册
- 原始发送
- 命令发送
- 数据发送

这一层的定位和前面的 `i2c_bus` 类似，目标不是只服务 LCD，而是为后续所有 `SPI` 外设提供统一入口。

### 4.2 LCD_ST7789V 驱动层

新增 `lcd_st7789v`，用于：

- 初始化 LCD 芯片
- 命令 / 数据写入
- 设置显示窗口
- 清屏
- 区域填充
- 字符和字符串绘制
- 字体缩放显示

当前版本先采用轻量字符点阵，优先保证模板结构清晰、易懂、可复用。

### 4.3 BSP LCD 板级适配层

新增 `bsp_lcd`，负责：

- `RST / PWR` 通过 `XL9555` 控制
- 板级 SPI 参数整理
- LCD 上电和复位时序
- 把板级参数组装成 `lcd_st7789v` 可用的配置结构

### 4.4 display_service 显示服务层

新增 `display_service`，负责：

- 缓存当前显示状态
- 首页重绘
- 版本号显示
- 阶段名显示
- LED 状态显示
- 蜂鸣器状态显示
- 最近按键事件显示

后续业务层如果要改显示内容，优先修改或调用这层，而不是直接去碰底层 LCD 驱动。

## 6. 本版本关键开发内容

### 6.1 为什么 SPI 也要做成模板层

如果这版只是为了 LCD 临时配一组 `SPI`，后面做：

- `TF 卡`
- 其他 `SPI` 屏
- `SPI` 传感器

就很容易重复造轮子。

所以这版和 `i2c_bus` 一样，把 `SPI` 先抽成通用访问层，再把 `LCD` 挂在它上面。

### 6.2 为什么 LCD 驱动名字要用 `lcd_st7789v`

如果只叫 `st7789v`，后面驱动一多时不够直观。  
改成带 `lcd` 前缀后，一眼就能知道它是显示驱动，不是别的外设控制器。

### 6.3 为什么需要 `bsp_lcd`

这块板子的 LCD 不是纯 ESP32 直连：

- `RST` 走 `XL9555 IO1_2`
- `PWR` 走 `XL9555 IO1_3`

所以：

- `lcd_st7789v` 不应该直接知道 `XL9555`
- 这些板级接线和控制时序应该收在 `bsp_lcd`

### 6.4 为什么显示服务要做缓存

当前 `display_service` 缓存了：

- 版本号
- 阶段名
- 最近一次按键事件
- 三路 LED 状态
- 蜂鸣器状态

这样业务层只负责更新状态，真正重绘由显示服务统一完成，结构更清楚，后续也更容易扩成脏区刷新。

### 6.5 为什么把显示格式改成 `SYS : OFF`

前面调试时发现用 `=` 号显示会受当前简化字模影响，屏上容易显示成异常字符。  
改成：

- `SYS : OFF`
- `BEEP : ON`

既更容易阅读，也更适合当前轻量字模。

## 8. 硬件前提

- LCD 驱动芯片：`ST7789V`
- 分辨率：`320 x 240`
- 接口模式：`SPI`
- `RST`：`XL9555 IO1_2`
- `PWR`：`XL9555 IO1_3`
- `MOSI`：`GPIO11`
- `DC/WR`：`GPIO13`
- `SCLK`：`GPIO12`
- `CS`：`GPIO21`

说明：

- 当前模板按“写屏为主”设计
- 当前版本把 `GPIO13` 按 `LCD DC/WR` 控制脚使用
- 如果后面模块接法不同，优先修改 `app_config.h`

## 9. 上板验证要点

### 9.1 基础显示链

- 上电后 LCD 能正常点亮
- 屏幕不是全白屏、全黑屏或花屏
- 能看到标题、版本号和阶段名

### 9.2 页面内容

- LED 状态区能显示三路 LED 模式
- 蜂鸣器状态区能显示 `BEEP : ON/OFF`
- 最近一次按键事件能刷新

### 9.3 按键联动

- `KEY0 ~ KEY2` 触发时，LED、蜂鸣器、LCD 同步变化
- `KEY3` 触发时，蜂鸣器状态区和最近事件区同步变化

### 9.4 启动日志

重点确认日志里能看到：

- `v1.4.0`

- `LCD ST7789V ready`
- `LCD board ready`
- `display service ready`
- `Event flow: XL9555 -> button_service -> unified_event_queue -> app_event_task -> led_service + beep_service + display_service`

## 12. 本版本意义

这一版的价值不只是“屏幕亮了”，更重要的是：

- 学会了如何把 `SPI` 也做成总线模板
- 学会了显示驱动如何分成“总线层 / 芯片层 / BSP层 / 服务层”
- 学会了让 LCD 成为统一事件链上的一个输出端
- 为后面显示：
  - 传感器值
  - 联网状态
  - AI 状态
  - 菜单页面

打下了清晰的模板基础
