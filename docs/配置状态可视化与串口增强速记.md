# 配置状态可视化与串口增强速记

## 这版在做什么

把 `v2.0.0` 的配置服务继续往前推进半步：

- 配置来源能直接显示
- 串口命令能直接看来源和状态
- 能手动测试当前 HTTP / OTA 地址是否有效

## 配置来源摘要

来源枚举：

- `DEFAULT`
- `NVS`
- `RUNTIME`

摘要文本：

- 全部来自默认值：`DEFAULT`
- 全部来自 NVS：`NVS`
- 全部是运行时临时修改：`RUNTIME`
- 混合状态：`MIXED`

## LCD 新增显示

项目区新增一行：

```text
CFG : DEFAULT
```

或：

```text
CFG : NVS
CFG : MIXED
```

## 新增串口命令

```text
cfg status
cfg source
cfg test http
cfg test ota
```

## 命令作用

### cfg status

- 打印当前配置
- 打印配置来源
- 打印 HTTP / OTA 当前状态

### cfg source

- 只看来源
- 适合快速确认当前值到底来自默认值还是 NVS

### cfg test http

- 使用当前 `http_test_url` 发起一次 HTTP 请求
- 用来快速验证配置改完后地址是否通

### cfg test ota

- 使用当前 `ota_version_url` 主动发起一次 OTA 版本检查
- 不依赖自动检查节奏

## 这版的价值

- 配置系统更可观察
- CLI 更适合调试
- 为后面的 `STA+AP + 网页配置` 打基础
