# 配置服务模板文档

## 1. 文档定位

这份文档把 `config_service` 整理成一份可复用模板，目标是：

- 给项目提供统一的运行时配置入口
- 把 NVS 读写和业务层解耦
- 让 `Wi-Fi / HTTP / OTA` 不再直接依赖写死宏值

---

## 2. 模块职责

`config_service` 这一层建议只负责：

- 初始化 NVS 配置存储
- 加载默认配置
- 从 NVS 读取保存配置
- 提供统一的配置查询接口
- 提供保存、恢复默认值接口

它不负责：

- Wi-Fi 联网本身
- HTTP 请求本身
- OTA 升级执行本身

也就是说，它只解决：

```text
配置从哪里来、如何保存、如何提供给其他服务
```

---

## 3. 推荐分层关系

建议保持这条结构：

```text
app_config.h 默认值
-> config_service
-> wifi_service / http_service / ota_service
```

这里的关键是：

- `app_config.h`
  - 负责默认值
- `config_service`
  - 负责运行时值

---

## 4. 推荐头文件模板

```c
#ifndef CONFIG_SERVICE_H
#define CONFIG_SERVICE_H

#include <stdbool.h>

#include "esp_err.h"

#ifdef __cplusplus
extern "C" {
#endif

typedef struct {
    char wifi_ssid[33];
    char wifi_password[65];
    char http_test_url[160];
    char ota_version_url[160];
} app_runtime_config_t;

esp_err_t config_service_init(void);
esp_err_t config_service_load(void);
esp_err_t config_service_save(void);
esp_err_t config_service_reset_to_default(void);
const app_runtime_config_t *config_service_get(void);
esp_err_t config_service_set_wifi(const char *ssid, const char *password);
esp_err_t config_service_set_urls(const char *http_url, const char *ota_url);
bool config_service_is_ready(void);

#ifdef __cplusplus
}
#endif

#endif
```

---

## 5. 推荐上下文结构

```c
typedef struct {
    bool inited;                  // 配置服务是否已初始化。
    bool loaded;                  // 当前配置是否已从默认值或 NVS 加载完成。
    app_runtime_config_t runtime; // 当前运行时配置缓存。
} config_service_ctx_t;
```

这个结构的重点是：

- 内存里始终保留一份“当前生效配置”
- 其他服务不需要关心数据来自默认值还是 NVS

## 6. 推荐默认值加载模板

```c
static void config_service_load_default(app_runtime_config_t *cfg)
{
    if (cfg == NULL) {
        return;
    }

    (void)snprintf(cfg->wifi_ssid, sizeof(cfg->wifi_ssid), "%s", APP_WIFI_STA_SSID);
    (void)snprintf(cfg->wifi_password, sizeof(cfg->wifi_password), "%s", APP_WIFI_STA_PASSWORD);
    (void)snprintf(cfg->http_test_url, sizeof(cfg->http_test_url), "%s", APP_HTTP_TEST_URL);
    (void)snprintf(cfg->ota_version_url, sizeof(cfg->ota_version_url), "%s", APP_OTA_VERSION_URL);
}
```

这段模板的作用是：

- 把 `app_config.h` 里的宏值转成运行时结构体默认值

## 7. 推荐初始化模板

```c
esp_err_t config_service_init(void)
{
    if (s_config.inited) {
        return ESP_OK;
    }

    esp_err_t ret = nvs_flash_init();
    if (ret == ESP_ERR_NVS_NO_FREE_PAGES || ret == ESP_ERR_NVS_NEW_VERSION_FOUND) {
        ESP_ERROR_CHECK(nvs_flash_erase());
        ret = nvs_flash_init();
    }

    if (ret != ESP_OK) {
        return ret;
    }

    config_service_load_default(&s_config.runtime);
    s_config.inited = true;
    return ESP_OK;
}
```

## 8. 推荐加载模板

```c
esp_err_t config_service_load(void)
{
    nvs_handle_t handle = 0;
    size_t len = 0;

    if (!s_config.inited) {
        return ESP_ERR_INVALID_STATE;
    }

    config_service_load_default(&s_config.runtime);

    esp_err_t ret = nvs_open("app_cfg", NVS_READWRITE, &handle);
    if (ret != ESP_OK) {
        s_config.loaded = true;
        return ESP_OK;
    }

    len = sizeof(s_config.runtime.wifi_ssid);
    if (nvs_get_str(handle, "wifi_ssid", s_config.runtime.wifi_ssid, &len) != ESP_OK) {
        // 没读到就继续使用默认值。
    }

    len = sizeof(s_config.runtime.wifi_password);
    if (nvs_get_str(handle, "wifi_pwd", s_config.runtime.wifi_password, &len) != ESP_OK) {
        // 没读到就继续使用默认值。
    }

    len = sizeof(s_config.runtime.http_test_url);
    if (nvs_get_str(handle, "http_url", s_config.runtime.http_test_url, &len) != ESP_OK) {
        // 没读到就继续使用默认值。
    }

    len = sizeof(s_config.runtime.ota_version_url);
    if (nvs_get_str(handle, "ota_url", s_config.runtime.ota_version_url, &len) != ESP_OK) {
        // 没读到就继续使用默认值。
    }

    nvs_close(handle);
    s_config.loaded = true;
    return ESP_OK;
}
```

## 9. 推荐保存模板

```c
esp_err_t config_service_save(void)
{
    nvs_handle_t handle = 0;
    esp_err_t ret = nvs_open("app_cfg", NVS_READWRITE, &handle);
    if (ret != ESP_OK) {
        return ret;
    }

    ESP_ERROR_CHECK(nvs_set_str(handle, "wifi_ssid", s_config.runtime.wifi_ssid));
    ESP_ERROR_CHECK(nvs_set_str(handle, "wifi_pwd", s_config.runtime.wifi_password));
    ESP_ERROR_CHECK(nvs_set_str(handle, "http_url", s_config.runtime.http_test_url));
    ESP_ERROR_CHECK(nvs_set_str(handle, "ota_url", s_config.runtime.ota_version_url));

    ret = nvs_commit(handle);
    nvs_close(handle);
    return ret;
}
```

## 10. 推荐查询接口模板

```c
const app_runtime_config_t *config_service_get(void)
{
    return &s_config.runtime;
}
```

这一步的价值很大，因为后面其他模块都可以统一写成：

```c
const app_runtime_config_t *cfg = config_service_get();
```

## 13. 恢复默认值模板

```c
esp_err_t config_service_reset_to_default(void)
{
    config_service_load_default(&s_config.runtime);
    return config_service_save();
}
```

这个接口后面非常有用，适合接：

- 调试菜单
- 按键恢复默认值
- 工厂重置

---

## 14. 常见注意点

- `NVS` 初始化要尽量早
- 第一次启动时，很多 key 本来就不存在，不要把它当失败
- 字符串长度要预留结尾 `\0`
- 写入后一定要 `nvs_commit()`
- 不建议每个服务都自己去 `nvs_open()`

如果配置里有 URL 字段，还建议多做两层保护：

- 保存前先做 URL 基础校验
- 启动加载 NVS 后，如果发现 URL 明显不合法，自动回退到 `app_config.h` 默认值

这样可以避免这种典型问题：

```text
cfg set http http://selftest/manual
-> cfg save
-> reboot
-> config_service_load() 从 NVS 读出坏 URL
-> http_service_process() 真去请求这个地址
-> getaddrinfo 失败
-> monitor 被错误日志刷满
```

所以配置服务模板里很推荐把“坏值回退默认值”直接当成固定能力保留。

---

## 15. 一句话记住

把 `config_service` 记成一句话就够了：

```text
它负责把默认配置、NVS 保存值和当前运行值统一成一层服务，
让其他模块只关心“当前配置是什么”，而不用关心“配置从哪来”。
```
