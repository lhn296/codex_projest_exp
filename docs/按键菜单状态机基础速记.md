# 按键菜单状态机基础速记

## 为什么这版先做菜单模板

目的不是做复杂 UI，而是先沉淀：

- 如何进入菜单
- 如何切换页面
- 如何选择项目
- 如何执行动作

## 继续复用现有按键链

当前建议继续使用：

```text
XL9555 -> button_service -> app_event_task -> menu_service
```

不额外增加独立按键。

## 最小菜单页面

- `MENU_HOME`
- `MENU_CONFIG`
- `MENU_ACTION`

## 最小按键规则

- `KEY0`：上一项
- `KEY1`：下一项
- `KEY2`：确认
- `KEY3`：返回 / 退出

## 建议入口

- `KEY3` 长按进入菜单

## 状态机核心

```text
隐藏
-> 进入菜单
-> 页面切换
-> 项目切换
-> 动作执行
-> 返回
-> 退出菜单
```

## 本版动作建议

- `RESET_CFG`
- `RELOAD_CFG`
- `REBOOT`

## 这版不要做什么

- 不做字符输入
- 不做编辑 URL
- 不做编辑 SSID
- 不做 LVGL
- 不做复杂动画

## 这版完成后会得到什么

- 一个可复用的菜单状态机模板
- 一个可复用的按键导航模板
- 一个可复用的菜单动作执行模板
