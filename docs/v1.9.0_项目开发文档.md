- 版本号：`v1.9.0`

## 2. 版本目标

本版重点不是只做版本判断，而是把下面这条链真正搭起来：

```text
Wi-Fi
-> 云端版本检查
-> 判断需要升级
-> 下载 firmware bin
-> 写入 OTA 分区
-> 设置启动分区
-> 重启进入新版本
```

本版目标主要包括：

- 处理 OTA 双分区空间问题
- 新增真实 OTA 下载与写分区能力
- 把 OTA 状态扩充到更完整的升级过程
- 让 LCD 和串口能同步显示升级阶段
- 为后面真实 bin 地址验证做准备

### 3.3 ota_service 接入真实 OTA 下载链

本版对：

- `components/services/ota_service.c`

当前这条函数链已经接上：

```text

-> esp_http_client_read
-> esp_ota_begin

```

这意味着：

- 当前项目已经不再只是“知道有新版本”
- 而是已经具备“真正把新固件写到 OTA 分区”的代码基础

### 3.4 真实下载采用边读边写方式

本版 OTA 真升级的关键学习点，是这条链：

```text
esp_http_client_read
-> esp_ota_write
```

也就是：

- 从云端一段一段读取固件数据
- 每读到一段就写到 OTA 分区
- 直到整个固件下载结束

这一步和前面 `HTTP / JSON` 读取链能衔接上，也让前面学到的内容真正进入了 OTA 正式场景。

- 项目版本切到 `v1.9.0`
- 阶段名切到 `v1.9.0 Real OTA Upgrade`
- 新增 `APP_OTA_WRITE_BUFFER_SIZE`

当前开发过程中，曾保留过一条很重要的安全策略：

- `APP_OTA_AUTO_UPGRADE = 0`

用于先把真实 OTA 主链接通，再避免一上电立刻误升级。

## 5. 本版本关键开发内容

### 5.1 为什么本版必须先处理分区表

因为真实 OTA 和前面的云端版本检查不一样。

版本检查只需要：

```text
联网
-> 请求 JSON
-> 比较版本
```

但真实 OTA 必须满足：

```text
当前运行分区之外，还要有另一个足够大的 OTA 分区可写
```

所以本版把分区表调整纳入主线，是必要动作，不是附带优化。

### 5.2 为什么保留 APP_OTA_AUTO_UPGRADE = 0

因为当前虽然真实 OTA 主链代码已经接好了，但云端接口里的：

- `url`

目前仍可能只是测试占位地址。

如果在这种情况下默认开启自动升级，就容易造成：

- 一上电直接进入失败升级流程
- 或误把测试环境当正式升级环境

所以本版选择先把：

```text
真实升级代码写好
但默认不自动触发
```

这样更稳。

### 5.3 为什么本版最终能完成真实升级闭环

因为在后续联调中，这两件关键事情都已经落地：

1. 云端接口中的 `url` 已换成真实可下载的七牛云 bin 地址
2. `APP_OTA_AUTO_UPGRADE = 1` 已开启并完成真实上板验证

当前实际验证链路是：

```text
腾讯云版本接口
-> 返回 version / url / message
-> url 指向七牛云真实 bin 地址
-> ESP32 自动下载写入 ota_1
-> 设置启动分区
-> 重启进入 v1.9.1
```

## 6. 关键代码归档

### 6.1 真实 OTA 升级入口

主要入口：

- `ota_service_download_and_apply()`

当前职责：

- 打开固件下载地址
- 获取目标 OTA 分区
- 开始 OTA 写入
- 循环读取固件并写入分区
- 完成后结束 OTA
- 设置新的启动分区
- 重启设备

### 6.2 目标分区选择

关键接口：

- `esp_ota_get_next_update_partition(NULL)`

当前作用：

- 找出“当前不在运行中的那个 OTA 分区”
- 把新固件写到它上面

### 6.3 OTA 写入链

关键接口：

- `esp_ota_begin()`
- `esp_http_client_read()`
- `esp_ota_write()`
- `esp_ota_end()`

这是本版最重要的一条链，因为它真正体现了：

```text
下载 + 写入
```

是怎么在 ESP-IDF 里结合起来的。

### 6.4 启动分区切换

关键接口：

- `esp_ota_set_boot_partition()`
- `esp_restart()`

这部分负责把：

```text
下载成功的新分区
```

真正变成下次重启后的运行分区。
