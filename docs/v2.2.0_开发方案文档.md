# v2.2.0 开发方案文档

## 版本主题

按键菜单基础模板版

## 版本目标

- 复用现有 `XL9555` 板载四按键
- 建立最小菜单状态机
- 建立最小菜单页面切换模板
- 支持查看关键配置摘要
- 支持执行少量基础动作

## 按键输入方案

本版不新增独立 GPIO 按键，直接复用当前已经跑通的：

```text
XL9555 -> button_service -> app_event_task -> display_service / menu_service
```

这样可以保持：

- 硬件接线不变
- 输入链不变
- 菜单逻辑只聚焦在“状态机模板”

## 菜单入口规则

建议使用：

- `KEY3` 长按进入菜单
- `KEY3` 短按返回
- 在一级菜单中 `KEY3` 再短按退出菜单

## 菜单按键分工

- `KEY0`：上一项
- `KEY1`：下一项
- `KEY2`：确认 / 执行动作
- `KEY3`：返回 / 退出

## 菜单页面规划

### 1. MENU_HOME

显示：

- 当前版本
- 当前配置来源
- 当前 Wi-Fi 状态

### 2. MENU_CONFIG

显示：

- `SSID`
- `HTTP URL` 简写
- `OTA URL` 简写

### 3. MENU_ACTION

显示：

- `RESET_CFG`
- `RELOAD_CFG`
- `REBOOT`

## 本版只做什么

- 菜单进入/退出
- 页面切换
- 菜单项选择
- 基础动作执行

## 本版不做什么

- 不做字符串编辑
- 不做输入法式文本输入
- 不做复杂层级菜单
- 不做 LVGL 图形界面
- 不做网页配置

## 模块建议

建议新增：

- `menu_service.c`
- `menu_service.h`

职责：

- 维护菜单状态机
- 维护当前页面和当前选中项
- 接收按键事件并更新菜单状态
- 通知 `display_service` 绘制菜单页

## 推荐状态划分

```text
MENU_STATE_HIDDEN
MENU_STATE_HOME
MENU_STATE_CONFIG
MENU_STATE_ACTION
```

## 推荐实现顺序

1. 先做菜单状态结构和键位规则
2. 再做菜单显示页面骨架
3. 再接入按键导航
4. 最后接 `RESET_CFG / RELOAD_CFG / REBOOT`

## 本版价值

这版的目标不是做最终产品菜单，而是沉淀：

- 菜单状态机模板
- 按键导航模板
- 菜单动作执行模板

为后续：

- `v2.3.0` STA+AP 网页配置
- 更后面的产品 UI / LVGL

提供一个轻量、稳定、可复用的参考基础。
