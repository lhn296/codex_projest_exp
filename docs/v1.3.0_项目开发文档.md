# v1.3.0 项目开发文档

## 1. 版本摘要

- 版本号：`v1.3.0`
- 日期：`2026-03-16`
- 平台版本：`ESP-IDF 5.5.3`
- 阶段主题：`XL9555 驱动基础版`
- 版本定位：完成 `I2C` 通用层、`XL9555` 驱动层与板级 `BSP` 适配层接入，把板载按键正式挂到现有事件架构中。

## 2. 版本目标

- 建立可复用的 `I2C` 访问层。
- 建立可复用的 `XL9555` 驱动层。
- 完成 `driver -> bsp -> services -> app` 四层结构演进。
- 接入板载按键、蜂鸣器、LCD 控制脚基础初始化能力。
- 保持上层按键状态机、统一事件队列、LED 业务链路不变。

## 3. 版本演进关系

- `v1.2.1`：重点是统一事件结构与任务通信。
- `v1.3.0`：重点转向板载外设接入与可复用驱动模板沉淀。

## 4. 本版本核心实现

### 4.1 驱动层

- 新增 `I2C` 通用访问层。
- 新增 `XL9555` 芯片驱动层。
- 对寄存器读写、引脚模式配置、端口缓存等关键逻辑增加了教学型注释。

### 4.2 BSP 层

- 新增 `bsp_xl9555` 板级封装。
- 接入板载按键 `KEY0~KEY3`。
- 接入蜂鸣器 `IO0_3`。
- 预留 LCD 控制口 `IO1_3`、`IO1_2`。
- 增加 `XL9555 INT` GPIO 输入与中断注册。

### 4.3 业务层与应用层

- `button_service` 继续复用已有按键状态机。
- `app_event_task` 继续复用统一事件消息处理链。
- `led_service` 保持原有灯效业务逻辑。

## 5. 软件结构归档

```text
components/
  app/
  bsp/
  driver/
  services/
  system/
```

分层职责：

- `driver`：通用驱动，不依赖当前板级接法。
- `bsp`：板级适配，负责把板上资源映射成统一接口。
- `services`：功能逻辑与状态机。
- `app`：任务编排、事件流转与系统初始化。

## 6. 关键硬件配置

- `XL9555 I2C 地址`：`0x20`
- `SDA/SCL`：`GPIO41 / GPIO42`
- `KEY0~KEY3`：`IO1_7 / IO1_6 / IO1_5 / IO1_4`
- `BEEP`：`IO0_3`
- `LCD 控制脚`：`IO1_3 / IO1_2`
- 当前 `INT` 调试接线：`XL9555 INT -> ESP32 GPIO39`

## 7. 关键问题记录

### 7.1 XL9555 INT 初期无响应

现象：

- 板载按键通过轮询可以正常识别。
- `irq wake btn=...` 日志不出现。
- 初期把 `INT` 转接到其他 GPIO 时，插拔排线能触发一次中断，但正常按键不会触发。

排查过程：

- 先通过“纯 INT 调试模式”确认问题是否被轮询兜底掩盖。
- 检查 `INT` 空闲电平、ISR 注册、中断触发方式与日志链路。
- 最终确认问题来自 `INT` 接线不稳定，而不是软件状态机或 ISR 注册逻辑。

结论：

- 当前 `INT` 已通过重新接线调通。
- 正式版本恢复为“中断优先 + 轮询兜底”模式，便于兼顾可靠性和调试性。

经验记录：

- `XL9555 INT` 这类信号线对接触可靠性较敏感。
- 上板调试时应优先确认：
  - 空闲电平是否稳定。
  - 中断线是否有可靠上拉。
  - 转接 GPIO 是否适合作为普通输入中断脚。
