# v2.0.0 项目开发文档

## 1. 版本摘要

- 版本号：`v2.0.0`
- 日期：`2026-03-20`
- 平台版本：`ESP-IDF 5.5.3`
- 阶段主题：`设备配置化基础版`
- 版本定位：把当前项目中的关键联网参数从“写死宏值”推进成“默认值 + NVS 保存值 + 运行时配置”的基础配置体系。

---

## 2. 本版目标

本版重点不是继续新增网络协议，而是把这些已经做好的能力：

- Wi-Fi
- HTTP
- OTA

从：

```text
直接读 app_config.h
```

升级成：

```text
默认值来自 app_config.h
-> 当前运行值由 config_service 管理
-> 可保存到 NVS
-> 重启后重新加载
```

---

## 3. 本版实现内容

### 3.1 新增 config_service

本版新增：

- `components/services/config_service.c`
- `components/services/config_service.h`

当前职责包括：

- 初始化 NVS
- 加载默认配置
- 从 NVS 读取配置
- 保存当前配置
- 恢复默认配置
- 提供统一配置查询接口

### 3.2 引入运行时配置结构

本版新增运行时配置结构：

- `app_runtime_config_t`

当前先纳入的字段有：

- `wifi_ssid`
- `wifi_password`
- `http_test_url`
- `ota_version_url`

这一步的意义是：

```text
把原来散落在宏定义里的关键参数统一成一份运行时配置
```

### 3.3 app_config.h 转成默认值来源

本版没有直接废掉 `app_config.h`，而是保留它作为：

```text
默认值来源
```

这样做的好处是：

- 保持当前工程结构稳定
- 后面继续做配置化时不需要推翻已有代码

### 3.4 主初始化顺序已调整

本版对：

- `components/app/app_main_task.c`

做了初始化顺序调整，当前主线变成：

```text
display_service_init
-> config_service_init
-> config_service_load
-> config_service_self_test
-> wifi_service_init
-> http_service_init
-> ota_service_init
```

核心原则是：

```text
先把配置读出来
再让联网服务开始使用这些配置
```

### 3.5 Wi-Fi / HTTP / OTA 已切到运行时配置读取

本版已经完成以下改造：

- `wifi_service`
  - 从 `config_service_get()` 读取 `SSID / password`
- `http_service`
  - 从 `config_service_get()` 读取 `http_test_url`
- `ota_service`
  - 从 `config_service_get()` 读取 `ota_version_url`

这意味着：

```text
当前联网参数已经不再直接依赖固定宏值
```

## 7. OTA 版本比较逻辑已修正

在本版联调过程中，已经顺手修正了一个很关键的问题：

### 7.1 原来的问题

之前 OTA 比较规则是：

```text
字符串不相等
-> 就认为存在更新
```

这会导致：

- 当前设备版本：`v2.0.0`
- 云端目标版本：`v1.9.1`

仍然进入：

- `READY`

这显然不符合真实升级判断逻辑。

### 7.2 当前修正方式

本版已经把 OTA 版本比较改成：

```text
v主版本.次版本.补丁版本
```

的数字比较方式。

当前已经验证：

- `v2.0.0` 对 `v1.9.1`
  - 结果：`NO_UPDATE`

这说明：

- 低版本不会再被误判成可升级目标

### 7.3 当前实际验证结果

上板日志已经确认：

```text
OTA_SVC: ota state -> IDLE msg=NO_UPDATE
OTA_SVC: ota no update, current=v2.0.0 target=v1.9.1
```

也就是说，本版当前已经同时验证了：

- 配置服务主链
- OTA 版本接口读取
- 版本比较逻辑修正

## 9. 串口配置入口已接入并验证

本版已经新增：

- `components/services/config_cli_service.c`
- `components/services/config_cli_service.h`

当前可在 `idf.py monitor` 中直接输入：

```text
cfg help
cfg show
cfg set wifi <ssid> <password>
cfg set http <url>
cfg set ota <url>
cfg save
cfg load
cfg reset
cfg reboot
```

当前已经验证通过的链路：

```text
cfg set ...
-> 修改运行时配置
-> cfg save
-> 写入 NVS
-> cfg reboot
-> 重启后重新 load
-> cfg show 仍可看到保存值
```

### 9.1 回显与提示符

当前串口 CLI 已支持：

- `cfg> ` 提示符
- 普通字符回显
- 退格回显

也就是说，当前 monitor 里已经可以比较自然地输入配置命令，而不是只能“盲打”。

### 9.2 坏 URL 保护

本版还顺手补了一个很重要的保护：

- 保存 URL 前做基础校验
- 启动加载 NVS 后，如果发现 URL 明显不合法，则自动回退到默认值

这一步就是为了避免出现：

```text
cfg set http http://selftest/manual
-> save
-> reboot
-> HTTP 持续请求坏地址
-> monitor 被错误日志刷满
```

当前这类情况已经被挡住了。
