# 阿里云函数计算搭建版本接口教程

> 适用阶段：`v1.8.0` 云端版本检查  
> 目标：在阿里云上快速搭一个能返回版本 `JSON` 的轻量接口，供 `ESP32` 访问

---

## 1. 这份教程的定位

这份教程只做一件事：

```text
搭一个最小可用的版本接口
-> 浏览器能访问
-> ESP32 能访问
-> 返回固定 JSON
```

当前重点不是学完整个阿里云函数计算体系，而是先把：

```text
版本检查接口
```

做出来。

---

## 2. 为什么阿里云函数计算也适合当前项目

对当前项目来说，它的优点是：

- 可以做轻量 HTTP 接口
- 不需要先维护一台完整服务器
- 后面也能继续扩到：
  - OTA 元数据接口
  - HTTP API
  - AI 中转接口

如果你后面更偏阿里云生态，它会是很顺的路线。

---

## 3. 当前准备内容

开始前建议准备：

- 一个阿里云账号
- 已完成实名认证
- 可以登录阿里云控制台
- 准备一段简单的版本 JSON

建议当前接口返回：

```json
{
  "version": "v1.8.1",
  "url": "https://your-domain/firmware/codex_project_tep_v1.8.1.bin",
  "message": "new firmware available"
}
```

---

## 4. 当前推荐的最小实现思路

当前建议走：

```text
函数计算
-> 创建一个 HTTP 函数 / Web 函数
-> 浏览器访问函数地址
-> 直接返回 JSON
```

这条路线的核心目标还是：

```text
先拿到一个稳定、可访问的版本接口 URL
```

---

## 5. 控制台操作建议顺序

### 第一步：进入函数计算控制台

在阿里云控制台进入：

- `函数计算 FC`

建议先选离你近的地域，例如：

- 华东
- 华北
- 华南

---

### 第二步：创建函数

建议创建一个名字类似：

- `codex-version-api`

当前优先选择：

- 支持 HTTP 访问的函数形态
- 在线编辑或 WebIDE 方式
- 能直接生成访问 URL 的方式

你在控制台里可能会看到：

- HTTP 触发器
- Web 函数
- 自定义域名 / HTTP 调用

对当前阶段来说，只要最后能得到一个浏览器可访问的 URL 就可以。

---

## 6. 建议的最小返回代码

如果你当前使用的是 JavaScript / Node.js 运行时，可以先按这种最小代码思路：

```javascript
module.exports.handler = async function (request, response, context) {
  const body = {
    version: "v1.8.1",
    url: "https://your-domain/firmware/codex_project_tep_v1.8.1.bin",
    message: "new firmware available",
  };

  response.setStatusCode(200);
  response.setHeader("content-type", "application/json; charset=utf-8");
  response.send(JSON.stringify(body));
};
```

这里的核心就是：

- 收到 HTTP 请求
- 直接返回固定 JSON

当前阶段不需要数据库，也不需要复杂业务逻辑。

---

## 7. 接口路径建议

如果平台允许你配置路径，建议接口就做成：

```text
/version
```

如果当前控制台给你的是一个函数 URL，而不是单独路径，也没关系。  
那就直接让这个 URL 本身返回版本 JSON。

最终对 ESP32 来说，只要：

```text
APP_OTA_VERSION_URL
```

能指向一个稳定返回 JSON 的 HTTPS 地址即可。

---

## 8. 浏览器验证标准

部署完成后，先在浏览器里验证接口。

期望看到：

```json
{
  "version": "v1.8.1",
  "url": "https://your-domain/firmware/codex_project_tep_v1.8.1.bin",
  "message": "new firmware available"
}
```

如果浏览器访问都不稳定，就先不要急着接入 ESP32。

---

## 9. ESP32 端如何接入

浏览器验证通过后，把工程里的：

```c
APP_OTA_VERSION_URL
```

改成阿里云函数计算实际生成的访问地址。

然后设备端主链就是：

```text
Wi-Fi
-> http_service 请求阿里云版本接口
-> ota_service 解析 version / url / message
-> LCD 显示 OTA 状态
```

---

## 10. 当前阶段最重要的检查点

你后面验证时，优先看这几点：

1. 浏览器能访问
2. 接口返回的是 `JSON`
3. 字段包含：
   - `version`
   - `url`
   - `message`
4. ESP32 当前网络环境能访问这个地址

最后一点仍然非常关键，因为云端接口“存在”不代表当前设备网络环境就一定能访问。

---

## 11. 当前最推荐的返回格式

建议长期固定：

```json
{
  "version": "v1.8.1",
  "url": "https://your-domain/firmware/codex_project_tep_v1.8.1.bin",
  "message": "new firmware available"
}
```

这样做的好处：

- 当前 `ota_service` 可直接复用
- 后面如果换云平台，设备端解析逻辑变化最小
- 后面进入真实 OTA 下载时，接口格式不用再重构

---

## 12. 当前不建议一开始就做的东西

这一步先不要急着做：

- 多环境部署
- 复杂权限控制
- 数据库存储版本记录
- 固件自动分发后台
- 设备平台化管理

先把：

```text
版本 JSON 接口
```

做稳定，已经足够推进当前版本。

---

## 13. 一句话记住

```text
阿里云函数计算也很适合当前阶段做“可直接返回版本 JSON 的轻量接口”，
先把版本检查主链跑通，再考虑后面的真实 OTA 下载与升级。
```
