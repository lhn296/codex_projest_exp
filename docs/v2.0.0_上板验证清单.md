# v2.0.0 上板验证清单

## 1. 本版验证目标

本版重点验证的是：

- `config_service`
- `NVS` 配置保存与读取
- 联网服务是否已成功切到运行时配置读取

不是这轮的重点：

- 自动 OTA 升级
- 新的云端协议
- 复杂配置入口

---

## 2. 上板前确认

请先确认：

- 当前编译版本是 `v2.0.0`
- `APP_OTA_AUTO_UPGRADE = 0`
- 默认 Wi-Fi 参数仍然可正常联网

---

## 3. 重点串口日志

### 3.1 配置服务初始化

应看到：

```text
CFG_SVC: config service ready
CFG_SVC: config loaded, ssid="..."
```

### 3.2 save/load 自检

应看到：

```text
CFG_SVC: config self-test start
CFG_SVC: config self-test passed
```

### 3.3 reset_to_default 自检

应看到：

```text
CFG_SVC: config self-test reset_to_default passed
```

### 3.4 恢复原始配置

应看到：

```text
CFG_SVC: config self-test restore done
```

---

## 4. 后续联网链验证

在配置服务日志通过后，还应继续确认：

- Wi-Fi 能进入 `GOT_IP`
- HTTP 请求仍然成功
- OTA 检查仍然能读取当前版本接口 URL

---

## 5. 这一版通过标准

满足以下条件即可判定本版第一轮通过：

- 配置服务初始化通过
- save/load 自检通过
- reset_to_default 自检通过
- restore 自检通过
- Wi-Fi 正常联网
- HTTP 正常
- OTA 版本接口读取正常
