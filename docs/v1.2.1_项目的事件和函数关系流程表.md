# 2. 本版核心变化

`v1.2.1` 的重点不是换任务结构，也不是加新外设，而是把：

```text
按键专用消息
```

升级成：

```text
统一事件消息
```

也就是把：

```text
button_service
-> app_button_msg_t
-> Queue
-> app_event_task
```

升级成：

```text
button_service
-> app_event_msg_t
-> Queue
-> app_event_task
```

# 3. 统一事件结构总览图

```mermaid
flowchart TD
    A["app_event_msg_t"] --> B[source]
    A --> C[type]
    A --> D[param1]
    A --> E[param2]
    A --> F[timestamp_ms]
```

可以这样理解：

- `source`
  - 事件来自哪个模块
- `type`
  - 事件属于什么类别
- `param1`
  - 主参数
- `param2`
  - 次参数
- `timestamp_ms`
  - 事件时间戳

# 4. 按键事件映射关系图

当前 `v1.2.1` 虽然还是按键事件，但已经按统一事件格式封装：

```mermaid
flowchart TD
    A["按键 SHORT/LONG/DOUBLE"] --> B[source = BUTTON]
    A --> C[type = BUTTON_EVENT]
    A --> D[param1 = button_id]
    A --> E[param2 = button_event]
    A --> F[timestamp_ms = now]
```

这意味着：

- 现在只服务按键
- 但结构已经为后续更多模块预留好了入口

# 6. 统一事件处理流程图

```mermaid
flowchart TD
    A["按键动作产生"] --> B["button_service 状态机识别事件<br/><sub>（btn_id 按键编号）</sub><br/><sub>（event 按键事件类型）</sub>"]
    B --> C["封装统一事件消息 app_event_msg_t<br/><sub>（source 事件来源）</sub><br/><sub>（type 事件类型）</sub><br/><sub>（param1 主参数）</sub><br/><sub>（param2 次参数）</sub><br/><sub>（timestamp_ms 时间戳）</sub>"]
    C --> D["xQueueSend"]
    D --> E["app_event_task 阻塞接收<br/><sub>（msg 统一事件消息）</sub>"]
    E --> F[解析 source type param1 param2]
    F --> G{是否为 BUTTON 事件?}
    G -- 是 --> H[按按键规则处理 LED]
    G -- 否 --> I[预留给后续模块]
    H --> J["调用 led_service"]
    I --> J
```

# 7. 主循环与任务关系图

```mermaid
flowchart LR
    A["app_main_task 主循环"] --> B["led_service_process"]
    A --> C["button_service_process"]
    D["app_event_task 独立任务"] --> E["xQueueReceive<br/><sub>（msg 统一事件消息）</sub>"]
    E --> F[统一事件分发]
    F --> G["app_event_handle_button_message<br/><sub>（msg 统一事件消息）</sub>"]
```

这个图最值得注意的点是：

- `app_main_task`
  - 负责周期性服务
- `app_event_task`
  - 负责阻塞式等待队列消息

# 8. 单击时序图

```mermaid
sequenceDiagram
    participant U as 用户按键
    participant ISR as bsp_button_gpio_isr_handler
    participant BTN as button_service_process
    participant Q as Queue
    participant EVT as app_event_task
    participant LED as led_service

    U->>ISR: 按下 触发下降沿
    ISR->>BTN: press_irq_pending = true
    BTN->>BTN: 消抖 + 状态机识别 SHORT
    BTN->>Q: button_service_publish_event(btn_id,event,now_ms)
    Q->>EVT: xQueueReceive(msg)
    EVT->>EVT: 统一事件分发
    EVT->>EVT: app_event_handle_button_message(msg)
    EVT->>LED: led_service_set_mode(led_id,mode)
```

