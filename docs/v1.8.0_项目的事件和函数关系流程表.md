# v1.8.0 项目的事件和函数关系流程表

## 1. 文档定位

这份文档用于说明 `v1.8.0` 的主链关系、初始化依赖、关键事件流和函数调用关系。

本版核心主题是：

```text
云端版本检查
```

也就是把 OTA 从“本地配置模拟”推进到：

```text
真实云端版本 JSON
-> HTTP 获取
-> OTA 解析版本信息
-> LCD 和日志显示结果
```

---

```mermaid
flowchart TD
    A["main [main.c]"] --> B["app_main [main.c]"]
```

## 10. LCD 联动图

```mermaid
flowchart LR
    A["wifi_service data"] --> D["display_service"]
    B["http_service data"] --> D
    C["ota_service data"] --> D
    D --> E["LCD home"]
```

说明：

- `wifi_service data` 主要包括：
  - `wifi state`
  - `ip`
  - `rssi`
  - `channel`
- `http_service data` 主要包括：
  - `http result`
  - `status code`
  - `message`
- `ota_service data` 主要包括：
  - `ota state`
  - `message`

建议本版 LCD 重点关注：

- `HTTP : OK or FAIL`
- `CODE : 200 or other`
- `OTA : READY or NO_UPDATE or FAIL`
- `MSG : new firmware available or other`

## 12. 当前版本最值得注意的实现点

### 12.1 当前主变化不在 Wi-Fi

这版不是重新学联网，而是基于已有 `Wi-Fi` 去接真实云端版本接口。

### 12.2 当前主变化也不在 HTTP 底层

HTTP 主链已经在 `v1.6.0` 打通，这版重点是让 `ota_service` 真正消费云端 JSON。

### 12.3 OTA 还没进真实下载链

当前版本最重要的是：

```text
真实版本检查
```

不是：

```text
真实固件升级执行
```
