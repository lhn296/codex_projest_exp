# v2.1.0 上板验证清单

## 验证目标

确认 `v2.1.0` 这版的三个重点已经成立：

- LCD 配置来源显示正常
- 串口配置增强命令正常
- OTA 手动检查入口正常

## 上板前确认

- 当前编译版本应为：
  - `v2.1.0`
- `APP_OTA_AUTO_UPGRADE = 0`
- `http_test_url` 默认可访问
- `ota_version_url` 默认可访问

## 基础启动验证

启动后确认日志里出现：

- `Project: codex_project_tep | Version: v2.1.0`
- `Stage: v2.1.0 Config Visualization & CLI Enhancement`
- `config service ready`
- `config cli ready, type 'cfg help' in monitor`

## LCD 验证

确认项目区能看到：

```text
CFG : DEFAULT / NVS / MIXED / RUNTIME
```

并且当前至少能正确显示：

- `CFG : NVS`

## CLI 命令验证

### 1. 配置来源

输入：

```text
cfg source
```

确认能输出：

- `summary=...`
- `wifi=...`
- `http=...`
- `ota=...`

### 2. 状态摘要

输入：

```text
cfg status
```

确认能输出：

- 当前配置
- 配置来源
- `http_ready / success / status / msg`
- `ota_state / has_update / msg`

### 3. HTTP 手动测试

输入：

```text
cfg test http
```

确认：

- 返回 `ret=0x0`
- `success=1`
- `status=200`

### 4. OTA 手动测试

输入：

```text
cfg test ota
```

确认：

- 返回 `ret=0x0`
- OTA 能进入 `CHECK`
- 如果云端版本更低，应回到：
  - `NO_UPDATE`

## 判定标准

如果以下条件都成立，则 `v2.1.0` 可判定通过：

- LCD 配置来源显示正常
- `cfg source` 正常
- `cfg status` 正常
- `cfg test http` 正常
- `cfg test ota` 正常
- OTA 版本比较没有误判低版本升级
