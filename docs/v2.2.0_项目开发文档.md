# v2.2.0 项目开发文档

## 版本主题

按键菜单基础模板版

## 本版目标

- 继续复用 `XL9555` 的四个板载按键
- 新增最小菜单状态机模板
- 建立菜单页面切换与基础动作执行链
- 为后续产品菜单和 `LVGL` 方案沉淀参考骨架

## 本版新增模块

- `menu_service.c`
- `menu_service.h`

职责：

- 维护菜单显示/隐藏状态
- 维护当前页面与动作页选中项
- 接收菜单模式下的按键事件
- 将菜单页内容交给 `display_service` 绘制

## 当前菜单页

### MENU HOME

显示：

- 当前版本
- 当前配置来源
- 当前 Wi-Fi 状态

### MENU CONFIG

显示：

- `SSID`
- `HTTP URL` 简写
- `OTA URL` 简写

### MENU ACTION

显示：

- `RESET_CFG`
- `RELOAD_CFG`
- `REBOOT`

## 当前按键规则

- `KEY3` 长按：进入菜单
- `KEY3` 短按：返回上一层 / 退出菜单
- `KEY0`：上一页或上一动作
- `KEY1`：下一页或下一动作
- `KEY2`：确认 / 执行动作

## 当前动作执行链

### RESET_CFG

```text
menu_service
-> config_service_reset_to_default()
-> config_service_load()
-> display_service_show_config_source(...)
```

### RELOAD_CFG

```text
menu_service
-> config_service_load()
-> display_service_show_config_source(...)
```

### REBOOT

```text
menu_service
-> esp_restart()
```

## 菜单显示链

```text
button_service
-> app_event_task
-> menu_service_handle_button_event()
-> menu_service_render()
-> display_service_show_menu_page()
-> display_service_process()
-> display_service_draw_menu_page()
```

## 上板验证结果

当前已经验证通过：

- `KEY3` 长按可进入菜单
- 菜单可正常退出
- `HOME / CONFIG / ACTION` 三页可切换
- `ACTION` 页的选项切换正常
- `RESET_CFG / RELOAD_CFG / REBOOT` 动作链可执行
- 菜单显示期间按键优先被菜单消费，不再继续触发原首页业务逻辑

## 当前已知情况

- 菜单显示效果还有少量细节可继续优化
- 但当前版本优先保证“菜单模板主链成立”
- 显示微调不作为这版阻塞项

## 本版结论

`v2.2.0` 已完成：

- 最小菜单状态机模板
- 最小页面切换模板
- 最小动作执行模板

这版的重点不是做完整产品 UI，而是为后续：

- 按键菜单扩展
- 具体项目产品菜单
- 未来 `LVGL` 界面方案

提供一个清晰、可运行、可复用的基础骨架。
